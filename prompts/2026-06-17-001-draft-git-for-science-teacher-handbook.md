The @gh-org-gitforsci/website repo is the book for this course. The purpose of the book is to share the course material as an open educational resource. I have built a skill to later on assess the course material for this. You have access to the skills through workspace permissions.

In this task, I want to draft the book. It is supposed to be as minimal as possible with checklists for educators to go through, so that a new educator can setup the course themselves. 

It should explain some of the pedagogy of this course. Pre-course work, hands-on exercises, side benefits (R, Quarto), live visual drawings in class (still need to develop these). 

Identify all software and accounts that are needed to run this class from a learner and teacher perspective. I am still working on generalizing the course material website in @gh-org-gitforsci-dev/website. You can look at the skill to better understand my concept of a three organization setup for twaching.

Instead of developing R scripts, I want the checklist to be more manual.

Outline the book for me. Consider using mermaid graphs to show how material belongs together. 

Make a detailed plan first. Ask me questions, spin out sub-agents if necessary.
